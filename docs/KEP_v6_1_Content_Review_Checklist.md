# KEP v6.1 — Content Review Checklist

Use this checklist before sharing KEP with students.

## Per question

- [ ] Question is clear.
- [ ] Four answer options are present.
- [ ] Correct answer matches one option.
- [ ] Explanation is useful.
- [ ] Difficulty is reasonable.
- [ ] Source note is added.
- [ ] Status is correct.
- [ ] Human reviewer checked it.

## Per subject

- [ ] Subject has enough questions.
- [ ] Questions are not all too easy.
- [ ] Explanations are consistent.
- [ ] Weak/duplicate questions are fixed.
- [ ] Resources exist for the subject.

## Before public testing

- [ ] Run Content Quality Dashboard.
- [ ] Test Practice.
- [ ] Test Mock.
- [ ] Test Progress.
- [ ] Ask 2–3 trusted testers for feedback.
