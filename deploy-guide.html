Production Checklist Release Notes

Deployment rule

KEP frontend can be deployed to GitHub Pages or Netlify. Supabase secret keys and AI keys must stay server-side only.

Use only the public Supabase URL and publishable/anon key in the browser.
Deployment paths

Choose one simple deployment method

Option A

GitHub Pages

  1. Extract the ZIP.
  2. Upload files to your GitHub repository.
  3. Go to repository Settings → Pages.
  4. Deploy from the main branch.
  5. Open the live GitHub Pages link.
  6. Open database-setup.html on the live domain and save Supabase URL/key again.
Option B

Netlify

  1. Extract the ZIP.
  2. Drag the folder into Netlify Deploys or connect GitHub.
  3. Keep publish folder as root.
  4. Open the Netlify URL.
  5. Open database-setup.html on Netlify and save Supabase URL/key again.
  6. Test student and admin journeys.
Supabase after deployment

Production settings to check

  • Add your live domain to Supabase Auth URL configuration.
  • Confirm RLS policies are enabled.
  • Keep service-role key private.
  • Use the public project URL, not the dashboard URL.
  • Use public publishable/anon key only in frontend.
