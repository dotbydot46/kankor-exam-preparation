Production checklist

Check KEP before inviting real students.

Use this checklist after deployment. It covers public pages, student flow, admin flow, content quality, Supabase, and AI safety.

Deployment Guide Quality Dashboard

Launch mode

Start with 2–3 trusted testers before sharing KEP widely.

Small testing first protects the project and students.
1. Public trust

Public website

  • Home page is clear and mobile friendly.
  • About page explains KEP mission.
  • Privacy and Terms pages are visible.
  • Contact details are updated from placeholders.
  • KEP does not claim official exam authority.
2. Student journey

Student app

  • Signup/login works on live domain.
  • Student Home opens correctly.
  • Practice loads published questions.
  • Mock Exam can start and finish.
  • Progress page shows attempts after tests.
3. Admin journey

Admin/reviewer

  • Admin/reviewer profile role is set.
  • Admin Review DB can load content.
  • Content Quality Dashboard loads.
  • AI suggestions are saved only for review.
  • Reviewer can accept/edit/reject suggestions.
4. Security

Database and AI safety

  • RLS is enabled on Supabase tables.
  • No service-role key is in frontend files.
  • No AI secret key is in frontend files.
  • Supabase Auth redirect URLs are configured.
  • Only admin/reviewer can use admin functions.
5. Content

Learning quality

  • Demo questions are replaced or labelled.
  • Subjects have enough published questions.
  • Weak explanations are improved.
  • Duplicate questions are checked.
  • Library resources have trusted source notes.
6. First testers

Controlled testing

  • Invite 2–3 trusted testers first.
  • Ask them to test on mobile.
  • Collect issues in one document.
  • Fix serious issues before wider sharing.
  • Review feedback weekly.
